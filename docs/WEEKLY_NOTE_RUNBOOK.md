# ランタンノート 週次運用マニュアル(B案: 週1レビュー → note予約投稿)

生成は毎日自動。人の手は「週1回・約15分のレビューと予約セット」だけ。
note公式の予約投稿を使うので、規約に触れず、アカウント凍結リスクもありません。

## 全体の流れ

```
毎朝9時   アプリが下書きを1本生成 → /articles に自動でたまる(週7本)
  ↓
週末(例:日曜) まとめてレビュー:読む → 承認 or 却下 → 本文コピー
  ↓
note      「予約投稿」で翌週7日分を各日にセット
  ↓
翌週      あなたは何もしなくても、毎日自動で公開される
```

## 前提(最初の1回だけ)

- **note プレミアム(月500円)への加入**。予約投稿はプレミアム限定機能です。
  加入しない場合は「週末に7本を手動公開(1本30秒)」でも運用できます。
- アプリ側の自動生成が動いていること(Render の `CRON_SECRET` / `OPENAI_API_KEY` 設定 + OpenAI残高)。

## 毎週やること(所要 約15分)

### 1. 下書きを開く
アプリの **Articles** ページを開く → 上部フィルタで **STATUS: Generated** を選ぶ。
その週にたまった下書き(最大7本)が並ぶ。

### 2. 1本ずつレビュー
各記事を開いて本文をチェック。判断は次の3つ:
- **良い** → 「Approve(承認)」を押す
- **手直しすれば使える** → note側で直す前提で「Approve」でOK(下書きは記録として残る)
- **イマイチ** → 「Reject(却下)」を押す(公開しない)

> レビューの目安: 事実の誤り・不自然な日本語・煽りっぽさが無いか。
> ランタンノートらしい「静かなトーン」から外れていたら却下でOK。

### 3. noteに下書き作成 + 予約セット
承認した記事について、noteで以下を繰り返す:

1. アプリの記事画面で **Title / Body / Tags のコピー**ボタンを使う
2. note の「投稿 → テキスト」で新規作成し、貼り付け
3. 見出し画像(ヘッダーやロゴを流用)を設定
4. タグを入力(コピーしたタグ)
5. 公開設定で **「予約投稿」** を選び、**日付を1日ずつずらして**セット
   (例: 月=記事A、火=記事B … 日=記事G)
6. 投稿時刻は毎日同じに固定(おすすめ **朝7:00**。通勤前に読まれやすい)

### 4. アプリ側のステータス更新(任意)
予約までしたら、アプリの記事を **「Mark Published」** にしておくと、
「どれを消化したか」がひと目で分かる。

## 運用のコツ

- **投稿時刻は固定**(毎朝7時など)。読者に「朝の習慣」として定着させる狙い。
- **7本揃わなくてもOK**。良い5本だけ予約 → 週5投稿でも十分。量より一貫性。
- **却下が続くようならトーン調整**。プロンプトを直せるので「もっと体験談風に」等あれば相談。
- **ストックを作る**。慣れたら2週間分まとめてレビューし、予約を2週先まで埋めると更に楽。
- 週次レビューを忘れないよう、note予約が切れる前に**日曜21時などにリマインドを設定**すると安心。

## 生成ペースを変えたいとき

- 現在: 毎日1本(週7本 / 月約30本 / OpenAI費用 月約$1)
- もっと選択肢が欲しい → GitHub の Actions → Article Pipeline → Run workflow で
  `count` を増やして手動生成、または cron の本数を戻す(相談を)。

## なぜ完全自動投稿にしないのか

noteには公式の投稿APIが無く、自動投稿はブラウザ自動操作(規約違反・凍結リスク・
パスワード預けの危険)しか手段がないため。note公式の「予約投稿」を使うこの方式が、
安全に「ほぼ自動」を実現する最善策です。
